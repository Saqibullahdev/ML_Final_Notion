üîπ Why did we use 4 neurons in the hidden layer?

There‚Äôs no fixed rule for the number of neurons in a hidden layer ‚Äî it depends on the complexity of the problem and experimenting.

But here are some common practices and reasons for choosing 4 neurons:

  1. Trial-and-error approach:
    • Start small (e.g. 3‚Äì5 neurons) and tune based on performance.
    • Too few neurons ‚Üí model underfits (can't learn enough)
    • Too many neurons ‚Üí model overfits (learns too much, poor generalization)
  1. Heuristic (rule of thumb):
    • A common heuristic is:

      Number of hidden neurons ‚âà (Number of input + output neurons) / 2

      ‚Üí (2 input + 1 output) / 2 = 1.5 ‚Üí round up ‚Üí try 3‚Äì5 neurons.

  1. Simplicity:
    • Since we only have 2 input features, we don‚Äôt need a very complex hidden layer.

‚úÖ So, 4 is a reasonable starting choice ‚Äî you can increase or decrease it during training.

üîπ Why use ReLU instead of Sigmoid in the hidden layer?

‚úÖ Reason we use ReLU:

  1. Avoid vanishing gradients:
    • Sigmoid squashes values between 0 and 1.
    • In deep networks, this leads to very small gradients, which slow down or stop learning (called vanishing gradient problem).
    • ReLU doesn‚Äôt have this issue for positive values.
  1. ReLU is faster:
    • It‚Äôs computationally simpler: ReLU(x) = max(0, x)
    • Speeds up training compared to sigmoid or tanh.
  1. ReLU allows sparsity:
    • It outputs 0 for negative values, which reduces complexity and activates fewer neurons, making the model faster and often more accurate.

‚ùóBut why not use sigmoid in hidden layer?

  • Sigmoid is still used in:
    • Output layer for binary classification (as in your case: pass/fail).
  • But in hidden layers, it's mostly avoided due to vanishing gradients and slower training.
