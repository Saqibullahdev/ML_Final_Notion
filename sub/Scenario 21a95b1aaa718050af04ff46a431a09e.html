Here’s a simple example to demonstrate anomaly detection using the Gaussian distribution method:

Scenario:

You're monitoring a server's health. You want to detect if the server is behaving abnormally. You collect two features:

  • x1x_1: CPU usage (%)
  • x2x_2: Memory usage (MB)

From normal operation, you collect the following data (just 5 examples shown):

ExampleCPU usage (%) (x1x_1)Memory usage (MB) (x2x_2)
115500
218520

🔢 Step 1: Estimate Gaussian Parameters

For each feature:

  • Mean μ\mu
  • Variance σ2\sigma^2

Let’s compute (rounded for simplicity):

  • μ1=16\mu_1 = 16, σ12=2\sigma_1^2 = 2 → for CPU
  • μ2=506\mu_2 = 506, σ22=100\sigma_2^2 = 100 → for memory

📉 Step 2: Compute Probability of New Example

Now, a new server reading comes in:

  • x=[30,800]x = [30, 800] → 30% CPU and 800MB memory

We compute the probability of this point under the Gaussian model:

p(x)=p(x1)⋅p(x2)p(x) = p(x_1) \cdot p(x_2)

Using the Gaussian formula:

p(x1=30)=12π⋅2e−(30−16)22⋅2≈very smallp(x_1=30) = \frac{1}{\sqrt{2\pi \cdot 2}} e^{-\frac{(30 - 16)^2}{2 \cdot 2}} \approx \text{very small}

p(x2=800)=12π⋅100e−(800−506)22⋅100≈very smallp(x_2=800) = \frac{1}{\sqrt{2\pi \cdot 100}} e^{-\frac{(800 - 506)^2}{2 \cdot 100}} \approx \text{very small}

So, p(x)p(x) will be extremely small.

🚨 Step 3: Compare with Threshold

If p(x)<ϵp(x) < \epsilon, label as anomaly.

Let’s say:

  • ϵ=0.0001\epsilon = 0.0001
  • Our p(x)=0.000000001p(x) = 0.000000001 (just for example)

So:

→ This is an anomaly!

✅ Conclusion:

  • The new point deviates significantly from normal data.
  • Anomaly detection flags it as abnormal server behavior.
